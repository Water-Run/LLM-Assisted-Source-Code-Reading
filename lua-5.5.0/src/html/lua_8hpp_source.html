5 * 这是 Lua 的 C++ 头文件包装器。Lua 核心库本身是用 C 语言编写的,可以同时
6 * 作为 C 和 C++ 代码编译。但是当在 C++ 项目中使用 Lua 时,需要告诉 C++ 编译器
7 * 这些函数使用的是 C 语言的链接约定(name mangling规则),而不是 C++ 的链接约定。
8 *
9 * 【为什么需要这个文件】
10 * - C++ 编译器会对函数名进行"名称修饰"(name mangling),以支持函数重载等特性
11 * - C 编译器不会进行名称修饰,函数名保持原样
12 * - 如果直接在 C++ 中包含 Lua 的 C 头文件,链接器会找不到函数(因为名称不匹配)
13 * - 使用 extern "C" 告诉 C++ 编译器:这些函数使用 C 链接约定,不要修饰名称
14 *
15 * 【使用方式】
16 * 在 C++ 项目中,包含这个文件而不是直接包含 lua.h:
17 * #include "lua.hpp" // 正确:在 C++ 中使用
18 * 而不是:
19 * #include "lua.h" // 错误:会导致链接错误
20 *
21 * 【文件结构】
22 * 1. extern "C" 块 - 声明 C 语言链接约定
23 * 2. 包含三个核心 Lua 头文件
24 ******************************************************************************/
